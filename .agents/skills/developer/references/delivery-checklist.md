# Developer Delivery Checklist

## Before Coding

- Confirm the problem and expected behavior.
- Check for existing patterns in the repo.
- Identify affected tests, schemas, and docs.

## During Coding

- Keep the change set focused.
- Preserve existing behavior unless the task explicitly changes it.
- Add or adjust tests for changed behavior.

## Before Handoff

- Summarize what changed.
- Record validation commands that were run.
- Call out unverified paths and follow-up work.
- Tell QA exactly what to validate.
- Tell security exactly what changed in the trust boundary, if relevant.
